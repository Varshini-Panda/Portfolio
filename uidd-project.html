Project Overview

The UIDD Change Process is a comprehensive web application built with Flask that automates the comparison of document revisions and identifies impacted automation scripts. This tool was developed to streamline the process of tracking changes in UIDD documents and assessing their impact on existing Robot Framework test scripts.

The application integrates with SVN repositories to fetch document revisions, generates HTML diffs to visualize changes, and uses keyword matching to identify which automation scripts are affected by specific document changes. This significantly reduces manual effort and improves accuracy in maintaining test scripts.

Key Features

  • SVN Integration: Automatically fetches document revisions from SVN repositories
  • Visual Diff Generation: Creates HTML-based difference visualizations with color-coded changes
  • Impact Analysis: Identifies impacted automation scripts through keyword matching
  • Multi-threading: Uses concurrent processing for efficient handling of multiple files
  • Excel Reporting: Generates detailed Excel reports of revision changes
  • Web Interface: User-friendly web interface for managing the change process

Technical Implementation

Architecture & Workflow

The application follows a client-server architecture with Flask handling the backend logic and serving the web interface. The workflow consists of the following steps:

User initiates the process through the web interface
Application connects to SVN repository to fetch latest revisions
Compares previous and current document versions using SVN commands
Generates HTML diff files highlighting changes with color coding
Extracts keywords from changed content using text processing algorithms
Matches keywords against script mapping database to identify impacted scripts
Identifies impacted automation scripts with confidence scoring
Generates Excel reports and updates tracking information for audit trail

def list_folders(repo_path):
    command = ['svn', 'list', '--depth', 'infinity', repo_path]
    result = subprocess.run(command, stdout=subprocess.PIPE, stderr=subprocess.PIPE, universal_newlines=True)
    if result.returncode == 0:
        items = result.stdout.split('\n')
        folders = [item.rstrip('/') for item in items if item.endswith('/')]
        return folders
    else:
        print("Error:", result.stderr)
        return []

3. Impact Analysis

This function identifies impacted scripts through keyword matching:

Impact Analysis Algorithm

def extract_robot_files(json_data, robotfilename):
    results = set()
    parameter_lower = robotfilename.lower()

    def recursive_search(data, key):
        if isinstance(data, dict):
            for k, v in data.items():
                if k.lower() == key:
                    if isinstance(v, dict):
                        for sub_key, sub_value in v.items():
                            results.update(sub_value)
                    elif isinstance(v, list):
                        results.update(v)
                recursive_search(v, key)
        elif isinstance(data, list):
            for item in data:
                recursive_search(item, key)

    recursive_search(json_data, parameter_lower)
    return results
